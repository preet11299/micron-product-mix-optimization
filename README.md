# Micron Product Mix Optimization
### Linear Programming Model — ISE 530: Optimization Methods for Analytics | USC Viterbi '26

---

## Overview

This project uses **linear programming (LP)** to determine the optimal allocation of Micron's semiconductor fab capacity across 8 products to maximize gross profit, subject to manufacturing and market constraints.

**Key result:** Shifting capacity toward HBM3E and DDR5 Server DRAM improves projected profit from **$13.95B → $16.80B (+20.4%)** while respecting all manufacturing and contractual limits.

---

## The Problem

Micron makes 8 products across DRAM and NAND with dramatically different margins:

| Product | Category | Gross Margin | Current Cap % |
|---|---|---|---|
| HBM3E | DRAM | 55% | 15% |
| DDR5 Server | DRAM | 38% | 25% |
| LPDDR5 Server | DRAM | 42% | 5% |
| LPDDR5 Mobile | DRAM | 35% | 12% |
| DDR5 PC | DRAM | 32% | 15% |
| NAND DC SSD | NAND | 28% | 15% |
| NAND Client SSD | NAND | 22% | 8% |
| NAND Mobile | NAND | 18% | 5% |

The question: **given fixed total fab capacity, what allocation maximizes profit?**

---

## The Model

**Decision variables:** xᵢ = % of total capacity allocated to product i (i = 1…8)

**Objective:**
```
Maximize Z = Σ (profit_per_capacity_i × x_i)
```

**Constraints:**
```
C1: Σ x_i = 100          (total capacity fully allocated)
C2: x_HBM ≤ 25           (manufacturing complexity limit)
C3: Σ x_DRAM ≥ 60        (market position)
C4: Σ x_NAND ≥ 15        (customer commitments)
C5: x_PC ≥ 8             (OEM relationships)
C6: x_Mobile ≥ 5         (smartphone OEMs)
C7: Σ x_DataCenter ≥ 35  (strategic priority)
C8: x_i ≤ demand_ceiling (cannot exceed market demand)
C9: x_i ≥ 0              (non-negativity)
```

Solved using **cvxpy** (ECOS solver). 8 variables, 9 constraint groups.

## Results

| Product | Current % | Optimal % | Change | Profit $B |
|---|---|---|---|---|
| **HBM3E** | 15% | **25%** | **+10%** | $7.33B |
| **DDR5 Server** | 25% | **32.5%** | **+7.5%** | $4.94B |
| LPDDR5 Server | 5% | 7.8% | +2.8% | $1.05B |
| LPDDR5 Mobile | 12% | 11.7% | −0.3% | $1.36B |
| NAND DC SSD | 15% | 15% | 0% | $1.26B |
| DDR5 PC | 15% | 8% | −7% | $0.85B |
| NAND Client SSD | 8% | 0% | **−8%** | $0 |
| NAND Mobile | 5% | 0% | **−5%** | $0 |

**Total: $13.95B → $16.80B (+$2.85B, +20.4%)**

### Why the model made these moves

- **HBM hits its ceiling immediately** — $0.293B profit per 1% capacity, nearly 2× the next best product
- **Server DRAM grows** — strong margins (38–42%) and AI-driven demand growing 30–56% YoY
- **Commodity NAND drops to zero** — 18–22% margins are the lowest in the portfolio; once contractual minimums are met, no additional allocation is justified

## How to Run

### Prerequisites
```bash
pip install cvxpy pandas numpy matplotlib
```

### Step 1 — Run the optimization FIRST
```bash
python3 optimization_model.py
```
Prints the optimal allocation table and constraint analysis to the console. Saves `results/optimal_allocation.csv`.

> ⚠️ You must run this before `visualizations.py` — the charts depend on `optimal_allocation.csv` which is generated here.

### Step 2 — Generate charts (only after Step 1)
```bash
python3 visualizations.py
```
Saves 4 charts to a new `visualizations/` folder: allocation comparison, profit distribution, margin/revenue bubble chart, and executive summary dashboard.
